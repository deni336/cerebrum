Viper Vision Troubleshooting, a Comprehensive Guide by Deni

This guide is meant to be a quick reference to assist in common issues seen with PoE and thermal applications.

Troubleshooting 101

  • Always verify and duplicate your problem. No troubleshooting can be done if the problem can’t be nailed down.

  • Try and see the problem first hand. If possible, remote in and look for common issues. For example, glass causing a reflection of heat, poor camera placement, etc…

  • Isolate the system down to its most basic form. For example, remove extra cameras and I/O’s.

  • Start taking notes. Write down IP addresses, temp readings, etc… from the beginning to help with future findings. This is a test and you will be quizzed on all parts of your troubleshooting process.

First steps

  1. Check for windows updates and that you are using the most current version of the software.
  2. Verify computer/camera IP addresses to ensure they are on the same subnet and that there are no conflictions. Tools that you can use to help with this is FLIR IP config, IPCONFIG and then ARP -a in Windows Command Prompt, EBUS Player and Viper Vision camera discovery.
  3. Check camera connection state via these steps: First, open Viper Vision and try to connect to the camera. Second, ping the camera in the Windows Command Prompt by tyiping ping 000.000.000.000(camera IP). Third, using EBUS, try to view the camera image. Last, open the cameras web server to view the image.
  4. On the camera web page, check the firewall and make sure it is off or that the RTSP and FLIRRESOURCE settings are on. Generally it is best to just turn off the Firewall.

More in depth

  • Check firmware of camera. Ensure it is the latest version.
  • Check for network driver updates.
  • Try substituting one component of the system. For example, swap cables, computers or camera.
  • Check that the switch used is not a managed switch. If it is, check the settings of the switch by entering 192.168.1.254 in a browser.

Scenarios

Not all encompassing, just common issues.

Incorrect Temperature Reading

  1. Save config, then clear and see if problem persists.
  2. Check for erroneous settings. For example, emitters, EBT, Gas detection, etc…
  3. Check emissivity and verify it is set to 1.
  4. Check transmissivity for proper setting per application. For example, cameras in enclosures will have a different setting than cameras in the open.

Adams Module/Input and Output

  1. Write down the needed I/O and alarm, if required.
  2. Delete I/O process.
  3. Verify IP of Adams module.
  4. Run Adam APAX utility and verify the I/O is functioning as advertised.
  5. Rebuild the I/O process.

UI problems

Generally when Viper Vision does not display the camera frames correctly, it can be fixed with the following steps.

  1. Close Viper Vision.
  2. At the top of the Windows File Explorer, click the view tab and check the hidden items box.
  3. Navigate to the following folder using Windows File Explorer. C:\Users\Viper\AppData\Roaming\Viper Imaging\ViperVision. Under Users, select the currently logged in Windows profile.
  4. Delete the ViperDockPanel.config file.
  5. Reopen Viper Vision.

Data Collection

Things that the software team will need to properly dissect issues.

  • Viper Vision logs. Can be found at C:\ViperConfigData\Logs.
  • Windows Error logs. Can be found by running Event Viewer and selecting windows logs then applications. On the right, use the filter current log option to select errors. Screenshot these errors for the same time frame.
  • Troubleshooting notes. Included in these notes should be system info, hardware info, IP addresses, camera count, etc…
  • Screenshot the IPCONFIG and ARP -a Windows Command Prompt readouts.
